# 응답 언어
- 모든 응답은 한국어로 진행한다.

# 환경 정보
- MacOS
- neovim version 0.12
- Iterm2 터미널 환경

# 파일 작성 규칙
- Neovim의 설정이 바뀌거나 lua/config/keymaps.lua 파일이 변경될 경우 README.md를 업데이트한다.
- lua 파일을 작성하라는 요청이 없을 경우 lua 파일을 작성하지 않는다.
- 설계를 할 경우 `CLAUDE_STRUCT.md` 파일에 작성한다.
- 계획을 할 경우 `CLAUDE_PLAN.md` 파일에 작성한다.
- 필요한 플러그인에 대한 리스트 `CLAUDE_PLUGINS.md` 파일에 정리한다.
- Plugins관련 파일을 추가하거나 삭제하면 CLAUDE_STRUCT.md, CLAUDE_PLAN.md, CLAUDE_PLUGINS.md 파일을 업데이트한다.

# neovim 기본 규칙
- 모든 단축키에 대한 설정은 config/keymaps.lua 파일에 작성한다.
- neovim에 관련된 기본 옵션 셜정은 config/options.lua 파일에 작성한다.

# neovim plugin 설정 규칙
- plugins에 있는 모든 플러그인파일에 주석으로 해당 파일의 이름을 작성한다.
- 주석으로 해당 플러그인에 대한 설명을 작성한다.
- 주석으로 해당 플로그인에 대한 사용법을 작성한다.
- 주석으로 해당 플러그인에 대한 기본 단축키 사용법을 작성한다.
- 플러그인 설정은 한번에 하나의 플러그인 설정만 진행한다.
- 플러그인의 기본 사용법이 불편하면 config/keymaps.lua에 편리한 단축키로 등록한다.
- config/keymaps.lua 파일에 키맵을 설정할 때 플러그인 별로 주석을 통해 구분한다.
- 플러그인 관련 단축키는 반드시 keymaps.lua 파일의 하단에 기록한다.
- 플러그인 단축키 구분 주석은 라인 밑에 작성한다.
  ```
  -- ############################################################################
  -- [[ Plugins ]]
  -- ############################################################################
  ```
- 각 플러그인 단축키는 아래 형식의 주석으로 구분한다.
  ```
  -- 플러그인이름 (설명)
  ```
- 플러그인 설정으로 불필요해진 설정은 제거한다.
