## 검색 / 치환 (단어 수정)

먼저 검색으로 대상을 찾고, 치환 명령이나 `cgn` 반복으로 단어를 수정한다.

| 단축키 / 명령           | 설명                                      |
|-------------------------|-------------------------------------------|
| `/단어` / `?단어`       | 앞/뒤 방향으로 단어 검색                  |
| `n` / `N`               | 다음 / 이전 검색 결과로 이동              |
| `*` / `#`               | 커서 아래 단어를 앞/뒤로 바로 검색        |
| `:s/old/new/`           | 현재 줄에서 첫 번째 `old`만 치환          |
| `:s/old/new/g`          | 현재 줄의 모든 `old` 치환                 |
| `:%s/old/new/g`         | 파일 전체의 모든 `old` 치환               |
| `:%s/old/new/gc`        | 파일 전체 치환 (각각 확인하며 y/n)        |
| `:%s/\<old\>/new/g`     | 단어 단위로 정확히 일치하는 것만 치환     |
| `:'<,'>s/old/new/g`     | (Visual 선택 후) 선택 범위만 치환         |

> 확인 치환(`gc`) 응답: `y` 적용 · `n` 건너뛰기 · `a` 나머지 전부 · `q` 중단.

**검색 후 `.`으로 반복 수정 (cgn 패턴)**

| 단축키       | 설명                                      |
|--------------|-------------------------------------------|
| `*`          | 커서 단어를 검색 대상으로 등록            |
| `cgn`        | 다음 매칭을 변경하고 새 단어 입력 후 `<Esc>` |
| `n` / `.`    | `n`은 건너뛰기, `.`은 동일하게 다시 수정  |

> 흐름: `*` → `cgn`으로 첫 수정 → `.` 반복(필요 시 `n`으로 건너뜀). 일괄이면 `:%s`가 빠르다.

## 프로젝트 전체 검색 / 치환

여러 파일에서 단어를 한 번에 바꾸려면, 먼저 검색 결과를 quickfix 목록에 모은 뒤 `cdo`/`cfdo`로 일괄 치환한다.

**1단계 — 검색 결과를 quickfix에 모으기**

| 단축키 / 명령              | 설명                                      |
|----------------------------|-------------------------------------------|
| `<leader>fg`               | Telescope 전체 텍스트 검색 (live grep)    |
| `<leader>fw`               | Telescope 커서 단어 검색                  |
| `<leader>cf`               | 커서 단어를 vimgrep으로 검색 → quickfix    |
| `:vimgrep /old/g **/*.lua` | 패턴을 직접 vimgrep으로 검색 (확장자 지정) |
| `<leader>co` / `<leader>cc`| quickfix 목록 열기 / 닫기                 |

> Telescope 결과 창에서 `<Ctrl-q>`를 누르면 검색 결과 전체가 quickfix로 보내진다.

**2단계 — quickfix 대상에 일괄 치환**

| 명령                            | 설명                                      |
|---------------------------------|-------------------------------------------|
| `:cfdo %s/old/new/g \| update`  | quickfix에 잡힌 각 **파일**에서 전체 치환 후 저장 |
| `:cdo s/old/new/g \| update`    | quickfix에 잡힌 각 **줄**에서 치환 후 저장 |
| `:cfdo %s/old/new/gc \| update` | 파일별로 확인(y/n)하며 치환 후 저장        |

> 흐름: `<leader>cf`(또는 `<leader>fg` → `<Ctrl-q>`)로 quickfix 채우기 → `:cfdo %s/old/new/g | update`.
> `| update`를 빼면 저장이 안 되니 주의. 단어 단위로 묶으려면 `\<old\>` 사용.

## 멀티커서 — 플러그인 (vim-visual-multi)

VSCode 스타일 멀티커서. 여러 위치에 커서를 놓고 동시에 편집한다.

| 단축키       | 설명                                      |
|--------------|-------------------------------------------|
| `<Ctrl-n>`   | 커서 단어 선택 / 반복 시 다음 매칭 추가   |
| `<Ctrl-Down>`| 아래로 커서 추가                          |
| `<Ctrl-Up>`  | 위로 커서 추가                            |
| `\\A`        | 현재 단어의 모든 매칭에 커서 생성         |
| `\\/`        | 정규식 검색 후 매칭 위치에 커서 배치      |
| `\\gS`       | 정규식 매칭 위치에 커서 생성              |
| `<Tab>`      | 커서 모드 ↔ 확장(Visual) 모드 전환        |
| `q`          | 현재 매칭 건너뛰고 다음으로 이동          |
| `Q`          | 현재 커서 제거                            |
| `n` / `N`    | 다음 / 이전 매칭으로 이동                 |
| `<Esc>`      | 멀티커서 모드 종료                        |

> 흐름: `<Ctrl-n>`으로 단어 잡고 → 반복으로 매칭 추가 → `c`/`i`/`a`로 동시 편집 → `<Esc>`.

## 멀티커서 — Vim 기본 (Visual Block)

`Ctrl-v`로 세로 블록을 선택해 여러 줄을 동시에 편집한다. 플러그인 없이 동작.

| 단축키          | 설명                                      |
|-----------------|-------------------------------------------|
| `<Ctrl-v>`      | 비주얼 블록 모드 진입                     |
| `<Ctrl-v>` + `j`/`k` | 위/아래로 블록 범위 확장             |
| `I`             | 블록 앞에 동시 입력 (`<Esc>` 시 전 줄 적용) |
| `A`             | 블록 뒤에 동시 입력 (`<Esc>` 시 전 줄 적용) |
| `c`             | 블록 영역 변경 후 동시 입력               |
| `d` / `x`       | 블록 영역 삭제                            |
| `r<문자>`       | 블록 영역을 한 문자로 일괄 치환           |
| `$` 후 `A`      | 길이가 다른 줄들의 끝에 동시 입력         |

> 흐름: `<Ctrl-v>` → `j`로 줄 선택 → `I`/`A`로 입력 → `<Esc>` 누르면 모든 줄에 반영.
